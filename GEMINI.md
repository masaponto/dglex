# Gemini CLI Guidelines

出力は必ず日本語で行ってください。

## 開発フローの強制
1. 実装を伴う Directive（指示）を受けた際、実装に着手する前に必ず「実装計画（Planning）」を提示し、ユーザーの承認を得ること。
2. 実装計画には以下の項目を含めること：
   - 変更の目的
   - 影響を受けるファイル
   - 具体的な変更内容の要約
   - テスト・検証戦略
3. ユーザーが「実行してください」またはそれに類する承認を出した後にのみ、ファイル操作（`write_file`, `replace` 等）を開始すること。

## 技術的標準
- DGL (Deep Graph Library) の作法に従い、NetworkX への変換や可視化の整合性を維持すること。
- コードの変更後は必ず `uv run pytest` を実行し、既存のテストが壊れていないことを確認すること。
